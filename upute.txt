Vrlo je bitno da se skupovi podataka nalaze u odgovarajućem folderu ("/content/sample_data", preporučeno u Google Colabu pošto on već radi s ovakvom folder strukturom) inače će odmah puknuti izvršavanje. Ukoliko se skupovi podataka tamo ne nalaze morat ćete promijeniti DATA_DIR varijablu unutar samog notebook-a.

Upute je generirao Gemini kao odgovor na upit:
Daj mi upute za pokretanje aplikacije unutar notebooka, slobodno navedi da ide ćeliju po ćeliju redom, analizirajući notebook

Odgovor:
UPUTE ZA POKRETANJE NOTEBOOKA

Notebook je namijenjen izvrsavanju u Jupyter ili Google Colab okruzenju.

Za pravilno pokretanje aplikacije potrebno je izvrsiti sve celije redom.

Postupak:

Otvoriti notebook u Jupyteru ili Google Colabu.

Po potrebi ucitati sve potrebne ulazne datoteke u radni direktorij notebooka.

U izborniku odabrati opciju:
Run -> Run all
(u Google Colabu: Runtime -> Run all)

Notebook ce se izvrsiti sekvencijalno, celiju po celiju, od vrha prema dnu.

Tijekom izvrsavanja pokrenut ce se priprema podataka, analiza te Flask aplikacija.

Nakon sto se dode do celije koja pokrece Flask server, izvrsavanje notebooka ce ostati aktivno dok je aplikacija pokrenuta.

Za ponovno pokretanje aplikacije potrebno je restartati runtime i ponovno odabrati Run all.

NAPOMENA

Notebook nije predviden za parcijalno izvrsavanje celija.
Pokretanje celija izvan redoslijeda moze dovesti do neispravnog rada aplikacije.
